# .clowalky/

This directory was created by `clowalky init`. It tells the [clowalky](https://github.com/n45os/claude-smart) orchestrator how to advance work in this project automatically — typically overnight, while your Claude Code 5-hour window is fresh.

## What's in here

- `README.md` — this file.
- `AGENT.md` — context any agent (Claude Code, Cursor, etc.) reads before touching `_plans/`.
- `_plans/` — one folder per multi-phase plan; folders prefixed `__` are archived (every row complete or deferred). See `AGENT.md` for the format.
- `_clwky_dev_plans/` — debug plans auto-generated by `clowalky reconcile` when STATUS.md drifts from git history. Created on demand; safe to delete once the drift is fixed.
- `skills/` — three agent-callable skills: `run-next-clowalky-phase` (runtime), `author-clowalky-plan` and `adopt-clowalky-plan` (authoring).
- `projects.local.json` — reserved for per-project orchestrator overrides; safe to ignore for now.

Clowalky currently runs in **dev mode** (the only mode for now). Dev mode treats STATUS.md as the durable record and reconciles it against git history when you run `clowalky reconcile`.

## Is it safe to commit?

Yes. `.clowalky/` is project metadata, not user secrets. Commit it so collaborators (and future-you on a different machine) see the same plans.

Removing this directory un-registers the project from clowalky's automation but does not affect your repo otherwise.
